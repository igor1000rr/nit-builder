# NIT Builder — создавай сайты на своём компьютере через локальную нейросеть

**TL;DR:** выложил в open source AI-конструктор сайтов, который работает на твоей видеокарте через LM Studio. Описываешь сайт словами — получаешь готовый HTML-файл за минуту. Без подписки, без облака, без интернета. MIT лицензия, 16 встроенных шаблонов, React Router v7, TypeScript.

**GitHub:** https://github.com/igor1000rr/nit-builder

---

## Проблема, которую я решал

Вокруг AI-генераторов сайтов сейчас тесно. Bolt, v0, Lovable, Readdy — все платные, все облачные, все требуют отправить твой запрос куда-то в Сан-Франциско. Для разработчика, который хочет быстро накидать лендинг клиенту, это нормально. Для человека из СНГ, который хочет свой сайт для маминой кондитерской, — не очень: 20 долларов в месяц за то, что может работать на его же RTX 3060.

Второй момент — все эти инструменты целятся в React. Генерят полноценные проекты с компонентами, импортами, сборкой. Это круто, если ты разработчик. Но для 80% задач — лендинг кофейни, портфолио фотографа, свадебное приглашение — React избыточен. Ты всё равно хочешь один HTML-файл, который можно кинуть на любой хостинг.

И третий — маленькие локальные модели (Qwen-Coder-7B, DeepSeek-Coder-6.7B) плохо справляются с генерацией полных React-проектов. Они путают импорты, забывают `"use client"`, генерят кривой Vite config. Но они **отлично** справляются с адаптацией готового HTML под конкретный контекст.

Так появилась идея.

## Как это работает

Архитектура максимально скучная и от этого надёжная:

```
Пользователь: "сайт для моей жены, она делает торты на заказ"
        ↓
[Planner LLM] → JSON: {
  business_type: "домашняя кондитерская",
  tone: "тёплый, семейный",
  sections: ["hero", "gallery", "about", "order", "contact"],
  color_mood: "warm-pastel",
  suggested_template_id: "handmade-shop"
}
        ↓
[Подгрузка шаблона handmade-shop.html из каталога]
        ↓
[Coder LLM] ← получает шаблон + JSON плана
           → стримит адаптированный HTML
        ↓
Live preview в iframe · Скачать одним файлом
```

Ключевое решение — **Planner видит список всех шаблонов прямо в system prompt** и сам выбирает подходящий через `suggested_template_id` в JSON-ответе. Никаких embeddings, векторных БД, Qdrant — LLM делает семантический матчинг через attention. Для 16 шаблонов 7B модель справляется идеально.

Второе ключевое решение — **Coder никогда не генерит с нуля**. Всегда получает готовый HTML-шаблон и адаптирует под план. Задача превращается из творческой ("придумай структуру сайта") в трансформационную ("замени тексты, скорректируй цвета, добавь/убери секции"). Это работает на Qwen-Coder-7B-Q4 с успешностью ~85%, когда from-scratch давала ~40%.

## Что умеет v1.0-beta

- **16 готовых шаблонов** под малый бизнес СНГ: кофейни, барбершопы, фотографы, портфолио разработчиков, свадьбы, фитнес, рестораны, репетиторы, маникюр, автосервисы, хендмейд, DJ, SaaS, стоматология, йога + универсальный fallback
- **3 LLM провайдера** с автоприоритетом: LM Studio (локально, бесплатно), Groq (облако, бесплатный tier), OpenRouter (платный)
- **Live preview** с переключением mobile/tablet/desktop
- **Polisher mode** — правки через чат: "сделай кнопки ярче", "добавь секцию с отзывами", "поменяй на фиолетовый"
- **SSE streaming** — видишь как сайт собирается по мере генерации
- **Скачивание одним HTML-файлом** — никакой сборки, хостишь где хочешь: GitHub Pages, Netlify, Cloudflare Pages, обычный хостинг

## Требования к железу

| Видеокарта | Модель | Скорость | Качество |
|---|---|---|---|
| 4 ГБ | Qwen2.5-Coder-3B-Q4 | Медленно | OK |
| **8 ГБ (RTX 3060/4060)** | **Qwen2.5-Coder-7B-Q4** | **Хорошо** | **Отлично** ⭐ |
| 12+ ГБ | Qwen2.5-Coder-14B-Q4 | Быстро | Превосходно |
| Нет GPU | Groq API (бесплатно) | Очень быстро | Отлично |

Если у тебя нет видеокарты — используешь Groq. Бесплатный tier даёт 14 тысяч запросов в день, для личного использования этого более чем достаточно. Если есть RTX 3060 и выше — запускаешь LM Studio, работает полностью локально, без интернета.

## Технологический стек

Не революция, но пара интересных решений:

- **React Router v7** (SSR). Я долго думал между Next.js App Router и React Router v7 — выбрал второй, потому что он проще, быстрее собирает, и file-based routing без скрытой магии
- **React 19** + **TypeScript strict**. Никаких `any`, никаких `@ts-ignore`, всё через Zod валидацию для runtime данных
- **Tailwind CSS v4** через Vite plugin. Новая версия, одна строка `@import "tailwindcss"` в CSS, всё работает
- **Vercel AI SDK** (`ai` + `@ai-sdk/openai`) для стриминга. Работает с любым OpenAI-совместимым провайдером (LM Studio, Groq, OpenRouter, vLLM, Ollama). Один клиент — три провайдера
- **Zustand? Redux?** Ничего. Весь state в `useState` и `useRef`. Проект достаточно простой, чтобы не тащить state manager.

Генерируемые сайты используют **Tailwind CDN** + **Alpine.js CDN**. Без сборки, без `npm install`, без webpack. Открываешь HTML в браузере — работает. Alpine покрывает 90% интерактивности (dropdown меню, табы, аккордеоны) в 15 килобайтах.

## Что у меня внутри по качеству

- **65 unit тестов** покрывают критичные модули: Zod схемы, SSE парсер, prompt sanitizer, templates catalog, CSRF, rate limit
- **0 TypeScript ошибок** в strict режиме
- **0 npm vulnerabilities** (`npm audit`)
- **GitHub Actions CI** гоняет typecheck + test + build на каждый push
- **Multi-stage Dockerfile** на Alpine, итоговый образ ~150 МБ
- **docker-compose.yml** — один `docker compose up` и у тебя локальный NIT Builder
- **Security**: CSRF (Origin/Referer), rate limiting sliding-window, guest daily limit, prompt injection filter, sandboxed iframe без `allow-same-origin`, CSP headers, HSTS в production

Общий объём кодбейза — около 5000 строк. Это примерно **в 5 раз меньше**, чем у моего предыдущего проекта с похожей идеей (nit.vibecoding.by — мультиагентный генератор с 7 ролями, RAG, fine-tuning). Я сознательно выкинул всё лишнее. Чем меньше кода, тем меньше багов, тем проще контрибьютить.

## Quick Start

```bash
git clone https://github.com/igor1000rr/nit-builder
cd nit-builder
npm install
cp .env.example .env
# Отредактируй .env: задай GROQ_API_KEY или LMSTUDIO_BASE_URL
npm run dev
```

Открываешь `http://localhost:5173`, пишешь "сайт для кофейни в Минске" — через минуту готовый HTML.

Для локального режима: скачиваешь [LM Studio](https://lmstudio.ai), в нём качаешь модель `Qwen2.5-Coder-7B-Instruct-Q4_K_M` (~4.5 ГБ), запускаешь локальный сервер (Developer → Start Server), и NIT Builder автоматически его подхватывает.

Для облачного режима: получаешь бесплатный ключ на [console.groq.com](https://console.groq.com/keys), задаёшь `GROQ_API_KEY=gsk_...` в `.env`. Всё.

## Как помочь проекту

Самый ценный вклад — **новые HTML-шаблоны**. Это не требует понимания AI-пайплайна, только навыки вёрстки. В `CONTRIBUTING.md` есть детальная инструкция: какие поля заполнить в каталоге, какие правила для шаблонов, как тестировать локально. Идеи для шаблонов, которых пока нет: тату-студия, цветочный магазин, языковая школа, бар/паб, агентство недвижимости, психолог, груминг для собак, ивент-фотограф, керамика, игровой разработчик, лендинг рассылки.

Второй — **отзывы и баг-репорты**. Если запустил и что-то не работает — открой issue с логами. Мне это нужно, чтобы понимать, какие модели лучше всего справляются с адаптацией шаблонов, какие промпты работают, какие нет.

Третий — **звёзды на GitHub**. Это звучит банально, но на ранних этапах open-source проекта каждая звезда реально влияет на видимость.
